Unity 三消项目 — 架构设计方案
答题方向：架构设计线
1. 文档目的
本文档指导 Demo 实现：按「数据 / 行为 / 表现 / 流程」划分模块，明确职责边界、依赖、排期与验收。对照参考视频补齐玩法与关键特效规格；
2. 玩法抽象（机制级）
2.1 核心循环
交换 → 校验 → 匹配 → 消除结算 → 下落补块 → 连锁 → 更新目标 → 胜负。特效/动画未完成前锁输入；Idle 后才可再操作。

2.3 Demo 范围与配置约定
纳入：三消闭环、行李箱收集、火箭/螺旋桨/炸弹、飘雪、消除/收箱/道具特效、Great 结算、基础音效
后置：双道具组合、冰/锁链、步数失败、多关、热更、埋点
目标数量：以 LevelConfig 为准；视频中可见更大数字（如 33），Demo 默认 15，可改配置对齐视频
棋盘尺寸：默认 9×8，写入配置，避免写死
彩球：视频可见；默认阶段 2 可选开关（Vfx/Booster 配置），人手紧可关
背景：一张暗色街景图即可，不单独立项做剧情 CG

4. 数据模块设计
4.1 BoardModel（运行时数据真值）
唯一棋盘真值：Cell 二维结构，提供 get/set、swap、clone、边界。纯 C#。View 只读镜像，不得私自改写。
4.2 TileType / BoosterType / ObjectiveType
枚举与静态表：普通四色、行李箱、空、火箭横/竖、螺旋桨、炸弹、彩球等。行为与表现都通过类型查表，避免魔法数。
4.3 LevelConfig（ScriptableObject）
rows/cols（默认 8/9）
objectiveType + objectiveCount（Demo 默认 15，可改为视频值）
initialBoard 布局
spawnWeights（仅普通块）
enableColorBall（彩球开关，默认 false 或按阶段打开）
4.4 VfxConfig / AudioConfig（ScriptableObject）
Vfx：飘雪发射率、碎屑数量、各特效时长、sortingOrder
Audio：交换/回弹/消除/收箱/道具/胜利 clip 引用与音量
5. 行为模块设计
5.1 MatchFinder
扫描横/竖匹配；识别 4 连、L/T、5 连；输出 MatchGroup + 可选道具生成信息。
5.2 SwapValidator
相邻校验；交换后无匹配则回滚。不负责播动画。
5.3 ClearResolver
清格；行李箱被波及则 Objective.Add(-1)；抛 ClearEvent / CollectEvent 给表现层。
5.4 GravitySystem / TileSpawner
列压缩；顶行按权重生成普通块（不刷行李箱）。输出 FallMove/SpawnMove。
5.5 CascadeResolver
状态机驱动匹配→消除→下落→补块。每步等待表现层完成回调。maxRounds=20。

读取 LevelConfig 目标数；维护 remaining；归零发 LevelWin。
6. 表现模块设计（动画 / 特效 / UI / 音效）
表现只订阅事件与命令队列，完成后回调流程模块。对照视频必做项如下。
6.1 BoardView / TileView（动画）
交换对滑、无效回弹
下落与顶上补块飞入（禁止凭空闪现）
选中高亮（轻微放大/描边）
6.2 AtmosphereFx（氛围）
开场即播，贯穿整关
全屏白色飘雪（大小/透明度变化）；可选细雨丝
不拦截点击；手机端可降粒子密度
6.3 ClearFx（消除）
短闪 → 碎裂 → 彩色碎屑（颜色跟块走）
可叠加星形闪点；时长可配置并纳入 Cascade 等待
6.4 CollectFx（收箱）
行李箱飞向顶栏图标 + 数字 punch
与普通碎裂区分，便于验收
6.5 BoosterFx（道具）
道具
表现要点
生成
锚点短聚光/合成后入格
火箭
行/列扫射拖尾，格依次碎裂
螺旋桨
飞行 + 星光拖尾，命中后清目标
炸弹
中心爆闪 + 3×3 碎裂
彩球（可选）
激活后同色块连锁清散
6.6 GoalHUD / WinPanel / SettleFx（UI）
顶栏：箱图标 + remaining
归零：Great + 星光爆闪 → 胜利页
不强制还原视频剧情角色 CG；背景用静态图
6.7 AudioController（音效）
交换成功/回弹、消除、收箱、道具激活、胜利
资源可后补，但接口与事件绑定本期就要留好（避免返工）
6.8 FxManager
统一收事件、排队列、回调解锁。禁止改 BoardModel。
7. 流程模块设计
7.1 LevelDirector
加载 LevelConfig；初始化 Board/Objective；启动 AtmosphereFx；组织开局。
7.2 InputController
手势→格坐标；非 Idle 直接丢弃输入。
7.3 GameFlowController
串联「输入→行为→等待表现→下一状态」；处理 LevelWin。
7.4 WinLoseHandler
胜利进入 SettleFx/WinPanel；本期无失败条件。
7.5 一次交换跨模块数据流
Input(流程) → Swap(行为) → Board(数据) → Cascade(行为) → Match/Clear/Gravity(行为) → Fx/View/Audio(表现) → Objective/HUD(数据事件+UI)

10. 实施顺序
10.1 阶段 0（约 1 人日）— 数据与骨架
工程/目录；BoardModel；Config 骨架；棋盘可见
可先挂 AtmosphereFx 验证不挡点击
10.2 阶段 1（约 2 人日）— 行为闭环 MVP
匹配、交换、下落、连锁、目标、HUD
DoD：不靠道具能收完目标；无 Error（消除可用占位闪白）
10.3 阶段 2（约 2 人日）— 表现对齐视频
飘雪、Clear/Collect、火箭/螺旋桨/炸弹（+ 可选彩球）、音效、Great
DoD：对照视频 0:00 飘雪、消除碎裂、道具轨迹、结算反馈
10.4 阶段 3 — 后置
双道具组合、冰/链、步数失败、多关、Addressables、埋点
